The Software Guild

The Software Guild is an immersive 12 week coding bootcamp. The majority of this bootcamp has a hands on feel which I love. We didn't just learn by reading chapters out of a curriculum, we learned by designing, implementing, and most importantly making mistakes!

I attended the Java cohort which focuses on Java backend Development. The first 6 weeks focused on Object Oriented Programming. The second 6 weeks focused on Data Driven Web Applications.

Week 1

  • Boolean Expressions and Controlling Program Flow
  • Debugging
  • Methods
  • Arrays
  • summative- ROCK,PAPER,SCISSORS


Week one was a crash course on basic programming concepts. We set up our IDE’s, issue tracking, and code review tools. After we were up and running we dove right into controlling program flow, boolean expressions, methods, arrays, and debugging. This week of the bootcamp significantly solidified prior programming concepts that I had self taught previously. The summative project for this week was to create a basic Rock, Paper, Scissors game. This project was intended to be in a single main class with optional methods. The outut is designed to be printed into the consol.



Week 2

  • Object-Oriented Concepts
  • Classes,Objects, and Creating New Types
  • References, Storage, and Memory Management
  • Interface, Composition, Inheritance
  • Data Marshalling and Unmarshalling
  • MVC and Tiered Application Design Concepts for CRUD
  • Software Development Lifecycle
  • sumative- AddressBook

Week two was jokingly referred to as the “face melter week” and it did not disappoint. This is where my work prior to the bootcamp really payed off. This week we worked through Object-Orientated Concepts, Classes, and Types, Memory Management, Composition and Inheritance, Data Marshalling, MVC and Tiered Application Design Concepts for CRUD, and Software Development Lifecycles.
Prior to this week I had a good understanding of how writing code works syntactically. As in I understood how what I had to write to complete a task, not the reason why we wrote the code in such ways. This week really opened my eyes to the OBJECT in OOP languages and working through Objects. We also got our first introduction to SOLID. Specifically the (**Separation of duties**) was a concept that I had been thinking over in my head on how developers executed.(**write MORE on this topic**)
The Summative assignment this week was to create our first CRUD console application, an Address Book. We first designed this program using ((***scratch paper techniques**)). Then we structured our program into the View, Controller, and Dao (data access object).

Week 3

  • unit testing (stateless and stateful)
  • Enums, JavaDateTime, BigDecimal
  • Lambdas, Streams
  • summative- Vending Machine

Week 3 was an intro into using libraries and JavaDocs. The way this course is set up is that as the weeks go by the instructors will “give” out less information and make the students look it up. So with the introduction to BigDecimal and JavaDateTime we then had to learn implentation through JavaDocs with some instructor assistance.
Alsong with this was our first introduction to unit testing. This is a powerful tool for testing methods as you build the application.
The summative assignment for this week was to create a virtual vending machine. This still follows a basic CRUD Application. This week included a service(business logic) layer in our design. We also had to include a full set of unit testing for our DAO and Service Layers. We are still using console outputs for this application, UX design comes after week 5.

Space Junk

Space Junk is a concept I designed for self teaching programming. It is a simple side scroller where the only goal is to pilot your rocket ship as long as possible while avoiding aestroids. I designed and implented this project using a trial by error approach. I had not yet completed much serious educaton in coding, I just knew that I wanted to complete the project.

All the code for this game is written using UnityScript, a similar dialect to JavaScript. I designed all of the sprites and graphics using an open source vector graphic editor. While this specific version is lacking in game content, I designed it so that I can quickly add features in the future.
