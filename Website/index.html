The Software Guild

The Software Guild is an immersive 12 week bootcamp that offers a unique backend development curriculum. The program hit the ground running hard and fast. The first 6 weeks was focused on object orientated principals and capped with a summative assessment. The last 6 weeks was focused on data driven web apps where we learned to sync databases and gather/retrieve data for display. The whole course then finished with both an individual and group summative assignment.

The majority of this bootcamp had a hands on feel which I loved. We would warm up most days with simple, but effective, coding challenges or thought problems. We didn't just learn by reading chapters out of a book, we learned by designing, implementing, and most importantly, making mistakes! This coding first mentality is the real winner in my book. When I applies with some knowledge of how to write code and graduated with a real understanding of how to create full stack applications.

Week 1 - Java Basics

  • Types and Variables
  • Boolean Expressions
  • Program Flow
  • Debugging
  • Methods
  • Arrays

My highlighted code for the week is the summative project for week 1. The assignment was to create a basic Rock, Paper, Scissors console game to play against a computer. The requirements stated that all code was to be contained inside a single class with optional additional methods.


Week 2 - Classes and Objects

  • Object-Oreinted Concepts
  • Classes and Objects
  • Memory Managment
  • Interfaces, Composition, and Inheritance
  • Collections and Maps

I want to highlight the controller for a DVD Library app I created. We were introduced to separating logical components of an app into individual classes. This controller is the logic behind the app and delegates responsibilities. Any user view or interaction is handled by the View, and and Data concern is handled by the DAO. This is a simple but effective way to write safe and clean code.


Week 3 - Intermediate Java

  • Service Layer
  • Unit Testing
  • Javadoc's
  • Libraries
  • Lambdas, Streams, and Aggregates
  • Enums

My code highlight for this week is the Service layer for our summative assessment. The project was to create a vending machine for the console that reads/writes from a file. This business logic layer is another step in separating concerns while writing applications. It has no concern of the inventory items, only that there is sufficient stock and money.

Week 12

Final Week! Week 12 was mostly a work week to finish the CMS with the team. Working as a group presented so many difficulties that were unseen with individual projects. We had to delegate duties, resolve merge conflicts, rely on others for deadlines, and more. I liked the project because of these difficulties. As a software developer I will run into these problems and need to be prepared.

The code I highlighted for this week is the DAO I created for uploading a static page. All created pages are stored in the database and displayed using the JSP. While not the hardest concept the logic, code, and implementation came easily. I proved the concept with a test project and delivered the working product.


Space Junk

Space Junk is a concept I designed for self teaching programming. It is a simple side scroller where the only goal is to pilot your rocket ship as long as possible while avoiding astroids. I designed and implemented this project using a trial by error approach. I had not yet completed much serious education in coding, I just knew that I wanted to complete the project.

All the code for this game is written using UnityScript, a similar dialect to JavaScript. I designed all of the sprites and graphics using an open source vector graphic editor. While this specific version is lacking in game content, I designed it so that I can quickly add features in the future.
